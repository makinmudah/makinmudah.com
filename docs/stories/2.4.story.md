# Story 2.4: SEO Implementation and Structured Data

## Status

Done

## Story

**As a** potential user searching for IT solutions or mentoring in Indonesia
**I want** Makin Mudah pages to appear prominently in search results with rich information
**So that** I can easily discover the services when searching for relevant terms

## Acceptance Criteria

1. All pages include unique, keyword-optimized meta titles: Landing (<60 chars), Applications catalogue, Mentoring catalogue
2. All pages include compelling meta descriptions (150-160 chars) with primary keywords and call-to-action
3. Open Graph meta tags implemented for social sharing: `og:title`, `og:description`, `og:image` (1200x630px), `og:url`, `og:type`
4. Twitter Card meta tags for enhanced Twitter sharing: `twitter:card`, `twitter:title`, `twitter:description`, `twitter:image`
5. Canonical URLs set for each page to prevent duplicate content issues
6. Language tag set to Indonesian (`<html lang="id">`)
7. FAQ structured data implemented using schema.org `FAQPage` markup with all 7 questions
8. Organization structured data added to landing page: name, logo, contact info, social profiles
9. Breadcrumb structured data added to catalogue pages showing: Home > Katalog > [Aplikasi/Mentoring]
10. XML sitemap generated listing all pages: `/`, `/katalog`, `/kontak` (future) with priority and change frequency
11. Robots.txt file created allowing all pages for indexing, reference to sitemap location
12. Structured data validated with Google Rich Results Test, no errors or warnings
13. Primary keywords targeted: "solusi IT mudah", "mentoring IT Indonesia", "aplikasi bisnis sederhana", "belajar IT terjangkau"

### SEO Implementation Patterns

**1. Meta Tags with useSeoMeta**

```vue
<script setup lang="ts">
const config = useRuntimeConfig()

useSeoMeta({
  title: 'Solusi IT Sederhana untuk Bisnis & Belajar - Makin Mudah',
  description:
    'Aplikasi bisnis UMKM dan program mentoring IT terjangkau. Bayar seperlumu, akses dimana saja, didampingi mentor. Buat hidup makin mudah!',
  ogTitle: 'Makin Mudah - Solusi IT Sederhana',
  ogDescription:
    'Aplikasi bisnis untuk UMKM dan mentoring IT dengan harga terjangkau. Tanpa instalasi, bisa diakses dimana saja.',
  ogImage: `${config.public.siteUrl}/images/og-image.jpg`,
  ogUrl: config.public.siteUrl,
  ogType: 'website',
  twitterCard: 'summary_large_image',
  twitterTitle: 'Makin Mudah - Solusi IT Sederhana',
  twitterDescription:
    'Aplikasi bisnis dan mentoring IT terjangkau. Bayar seperlumu, akses dimana saja.',
  twitterImage: `${config.public.siteUrl}/images/og-image.jpg`,
})
</script>
```

**2. Structured Data with JSON-LD**

```vue
<script setup lang="ts">
// FAQ Structured Data
const faqSchema = {
  '@context': 'https://schema.org',
  '@type': 'FAQPage',
  mainEntity: [
    {
      '@type': 'Question',
      name: 'Berapa sih biayanya?',
      acceptedAnswer: {
        '@type': 'Answer',
        text: 'Kami pakai sistem "Bayar Seperlumu, Bayar Semaumu". Untuk aplikasi bisnis mulai dari Rp 99.000/bulan...',
      },
    },
    // ... more questions
  ],
}

useHead({
  script: [
    {
      type: 'application/ld+json',
      children: JSON.stringify(faqSchema),
    },
  ],
})
</script>
```

**3. Organization Schema**

```typescript
const organizationSchema = {
  '@context': 'https://schema.org',
  '@type': 'Organization',
  name: 'Makin Mudah',
  url: 'https://makinmudah.com',
  logo: 'https://makinmudah.com/images/logo.png',
  description: 'Solusi IT sederhana untuk bisnis dan belajar',
  contactPoint: {
    '@type': 'ContactPoint',
    contactType: 'Customer Service',
    email: 'support@makinmudah.com',
  },
  sameAs: [
    // Social media URLs (if available)
  ],
}
```

**4. Breadcrumb Schema**

```typescript
// For /katalog page
const breadcrumbSchema = {
  '@context': 'https://schema.org',
  '@type': 'BreadcrumbList',
  itemListElement: [
    {
      '@type': 'ListItem',
      position: 1,
      name: 'Home',
      item: 'https://makinmudah.com',
    },
    {
      '@type': 'ListItem',
      position: 2,
      name: 'Katalog',
      item: 'https://makinmudah.com/katalog',
    },
  ],
}
```

### Sitemap Configuration

**Using @nuxtjs/sitemap Module:**

```typescript
// nuxt.config.ts
export default defineNuxtConfig({
  modules: ['@nuxtjs/sitemap'],

  sitemap: {
    hostname: 'https://makinmudah.com',
    gzip: true,
    routes: [
      {
        url: '/',
        changefreq: 'weekly',
        priority: 1.0,
      },
      {
        url: '/katalog',
        changefreq: 'monthly',
        priority: 0.8,
      },
    ],
  },
})
```

**Manual sitemap.xml (if not using module):**

```xml
<?xml version="1.0" encoding="UTF-8"?>
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
  <url>
    <loc>https://makinmudah.com/</loc>
    <changefreq>weekly</changefreq>
    <priority>1.0</priority>
  </url>
  <url>
    <loc>https://makinmudah.com/katalog</loc>
    <changefreq>monthly</changefreq>
    <priority>0.8</priority>
  </url>
</urlset>
```

### Canonical URLs

**Purpose:** Prevent duplicate content issues, consolidate link equity

**Implementation:**

```vue
<script setup lang="ts">
const config = useRuntimeConfig()
const route = useRoute()

useSeoMeta({
  // ... other meta tags
  canonical: `${config.public.siteUrl}${route.path}`,
})
</script>
```

**Note for /katalog page:**

- Hash fragments (#aplikasi, #mentoring) don't create separate pages for SEO
- Single canonical: `https://makinmudah.com/katalog`
- No need for separate canonicals per hash

[Source: Google Search Central, canonical URL guidelines]

### SEO Monitoring & Optimization

**Metrics to Track (via Google Search Console):**

- **Impressions**: How often pages appear in search results
- **Clicks**: How many users click through to site
- **CTR**: Click-through rate (optimize with better titles/descriptions)
- **Average Position**: Target top 10 (page 1) for primary keywords
- **Coverage Issues**: Fix any indexing errors

**Ongoing Optimization:**

1. **A/B Test Meta Descriptions**

   - Search for primary keywords

   - Keep changefreq accurate in sitemap
   - Add new products/services as they launch

   - Guest posts on Indonesian tech blogs
   - Partnerships with UMKM associations
   - Social media sharing
