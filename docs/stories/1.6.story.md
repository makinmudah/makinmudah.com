# Story 1.6: Final CTA Section and Footer

## Status

Draft

## Story

**As a** visitor,
**I want** a clear call-to-action after reading the full landing page,
**so that** I can easily take the next step to explore solutions.

## Acceptance Criteria

1. Final CTA section displays headline "Siap untuk hidup yang makin mudah?" (centered, large, navy color)
2. CTA button "Intip Solusinya" displayed below headline (teal background, white text, large size)
3. CTA button navigates to `/catalogue-apps` (consistent with earlier CTAs)
4. Section has subtle background color or pattern to make it visually distinct
5. Button is accessible (44x44px min, keyboard navigable, ARIA label if icon-only)
6. Footer section displays below CTA with light gray background
7. Footer contains copyright notice: "© 2026 Makin Mudah. All rights reserved." (or similar)
8. Footer text is small (12-14px), centered, gray color
9. Footer has minimal padding (16-24px vertical)
10. Footer is responsive and looks clean on all screen sizes

### Implementation Recommendations

**Recommended Approach:** Simple presentational components

**FinalCta Component:**

- Use `<NuxtLink>` for navigation button (automatic prefetching and SPA navigation)
- Apply Tailwind utility classes directly in template
- No complex state management needed
- Consider subtle animation on button hover (scale or shadow change)

**AppFooter Component:**

- Minimal markup: `<footer>` with copyright text
- Could be made globally available via Nuxt layouts in future stories
- For now, import directly in landing page

**Background Pattern Options for Final CTA:**

- Option 1: Subtle gradient (`bg-gradient-to-r from-gray-50 to-white`)
- Option 2: Light teal tint (`bg-teal-50`)
- Option 3: Geometric pattern using Tailwind or CSS background image
- Recommendation: Use subtle gradient or light teal tint for MVP simplicity

### Accessibility Implementation

**Keyboard Navigation:**

- Tab key: Navigate to CTA button
- Enter or Space: Activate button (navigate to catalogue)
- Shift+Tab: Navigate backwards

**ARIA Attributes:**

- Button should have descriptive text (already satisfied with "Intip Solusinya")
- No additional ARIA labels needed if text is clear
- Use semantic `<button>` or `<NuxtLink>` element (NuxtLink renders as `<a>`)

**Focus Indicators:**

- Visible outline or ring on focused button (`focus:ring-2 focus:ring-teal-500`)
- Sufficient contrast for visibility (teal ring on white/gradient background)

**Screen Reader Support:**

- Headline uses semantic `<h2>` element
- Button text is descriptive and context-appropriate
- Footer uses semantic `<footer>` element

**Color Contrast (WCAG AA):**

- Navy headline (#1E3A8A) on white: >7:1 ratio ✓
- Teal button (#14B8A6) with white text: >4.5:1 ratio ✓
- Gray footer text (#4B5563) on light gray (#F3F4F6): >4.5:1 ratio ✓

[Source: docs/ui-architecture.md#8-styling-guidelines - Color palette]
