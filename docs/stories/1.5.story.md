### Component Specifications

**Component Location:** `components/landing/FaqAccordion.vue`

[Source: docs/ui-architecture.md#3-project-structure]

**Component Template Structure:**

```vue
<script setup lang="ts">
// ============================================================================
// IMPORTS
// ============================================================================
import { ref } from 'vue'
import { Disclosure, DisclosureButton, DisclosurePanel } from '@headlessui/vue'
import { ChevronDownIcon } from '@heroicons/vue/24/outline'

// ============================================================================
// TYPES
// ============================================================================
interface FaqItem {
  question: string
  answer: string
}

// ============================================================================
// DATA
// ============================================================================
const faqItems = ref<FaqItem[]>([
  {
    question: 'Berapa sih biayanya?',
    answer: '...content from docs/faq-content.md...',
  },
  // ... 6 more items
])

// ============================================================================
// STATE
// ============================================================================
const openIndex = ref<number | null>(null)

// ============================================================================
// METHODS
// ============================================================================
function toggleItem(index: number) {
  openIndex.value = openIndex.value === index ? null : index
}
</script>

<template>
  <section class="bg-white py-12 md:py-16 lg:py-20">
    <div class="container mx-auto px-4">
      <h2 class="mb-8 text-center text-3xl font-bold text-navy-900 md:mb-12 md:text-4xl">
        Pertanyaan yang Sering Ditanyakan
      </h2>

      <div class="mx-auto max-w-3xl space-y-4">
        <Disclosure
          v-for="(item, index) in faqItems"
          :key="index"
          v-slot="{ open }"
          :default-open="false"
        >
          <DisclosureButton
            class="flex w-full items-center justify-between rounded-lg bg-gray-50 px-6 py-4 text-left text-lg font-semibold text-navy-900 transition-colors hover:bg-gray-100 focus:outline-none focus:ring-2 focus:ring-teal-500 md:text-xl"
            @click="toggleItem(index)"
          >
            <span>{{ item.question }}</span>
            <ChevronDownIcon
              :class="open ? 'rotate-180 transform' : ''"
              class="h-6 w-6 text-teal-600 transition-transform duration-300"
            />
          </DisclosureButton>

          <Transition
            enter-active-class="transition duration-300 ease-out"
            enter-from-class="opacity-0 -translate-y-2"
            enter-to-class="opacity-100 translate-y-0"
            leave-active-class="transition duration-200 ease-in"
            leave-from-class="opacity-100 translate-y-0"
            leave-to-class="opacity-0 -translate-y-2"
          >
            <DisclosurePanel class="px-6 pb-4 pt-2 text-base text-gray-700 md:text-lg">
              <div v-html="item.answer" />
            </DisclosurePanel>
          </Transition>
        </Disclosure>
      </div>
    </div>
  </section>
</template>
```

[Source: docs/ui-architecture.md#4-component-standards]

**Alternative: Native `<details>`/`<summary>` Implementation:**

If avoiding HeadlessUI dependency, use native HTML with custom accordion behavior:

```vue
<script setup lang="ts">
const openIndex = ref<number | null>(null)

function handleToggle(index: number, event: Event) {
  event.preventDefault()
  openIndex.value = openIndex.value === index ? null : index
}
</script>

<template>
  <details
    v-for="(item, index) in faqItems"
    :key="index"
    :open="openIndex === index"
    @toggle.prevent="handleToggle(index, $event)"
  >
    <summary class="...">{{ item.question }}</summary>
    <div class="...">{{ item.answer }}</div>
  </details>
</template>
```

### Accessibility Implementation

**Keyboard Navigation:**

- Tab key: Navigate between question headers
- Enter or Space: Toggle expand/collapse
- Shift+Tab: Navigate backwards

**ARIA Attributes (if not using native `<details>`):**

- `aria-expanded="true|false"` on question button (indicates expanded state)
- `aria-controls="panel-{index}"` on question button (links to answer panel)
- `role="region"` on answer panel (identifies as accordion panel)
- `aria-labelledby="button-{index}"` on answer panel (associates with question)

**Focus Indicators:**

- Visible outline or ring on focused elements (`focus:ring-2 focus:ring-teal-500`)
- Sufficient contrast for visibility (teal on white/gray backgrounds)

**Screen Reader Support:**

- Questions use semantic `<button>` or `<summary>` elements
- Clear announcement of expanded/collapsed state
- Answer content properly associated with question

**Color Contrast (WCAG AA):**

- Navy questions (#1E3A8A) on white: >7:1 ratio ✓
- Gray answers (#374151) on white: >4.5:1 ratio ✓
- Teal icon (#14B8A6) on gray: >3:1 ratio for large elements ✓

### Implementation Recommendations

**Recommended Approach:** Use HeadlessUI Vue `Disclosure` component

**Rationale:**

- Already in project dependencies (zero additional bundle cost)
- WCAG 2.1 AA compliant out of the box
- Handles keyboard navigation automatically
- Provides unstyled primitives (full Tailwind styling control)
- Well-maintained and documented

**Alternative Approach:** Native `<details>`/`<summary>` elements

**Pros:**

- Zero JavaScript dependency
- Browser-native accessibility
- Semantic HTML

**Cons:**

- Limited styling flexibility (especially for animations)
- Inconsistent browser behavior for "one open at a time" accordion
- Requires custom JavaScript for accordion behavior

**Recommendation:** Use HeadlessUI unless bundle size becomes critical concern.

### Performance Considerations

**Animation Performance:**

- Use CSS transitions (GPU-accelerated) instead of JavaScript animations
- Avoid animating `height` directly (causes layout reflow)
- Use `transform` and `opacity` for smooth transitions

**Bundle Impact:**

- HeadlessUI Disclosure: ~2KB additional (already included in project)
- ChevronDownIcon from Heroicons: ~1KB
- Total component overhead: <5KB gzipped

**Layout Shift Prevention:**

- Questions have fixed height (padding determines tap target)
- Answers expand/collapse smoothly without affecting other sections
- No CLS (Cumulative Layout Shift) when toggling

[Source: docs/ui-architecture.md#2-frontend-tech-stack - Performance targets]
