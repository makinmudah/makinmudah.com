# Story 1.3: Value Propositions Section with Three-Column Layout

## Status

Done

## Story

**As a** visitor,
**I want** to quickly understand the three main benefits of Makin Mudah,
**so that** I can decide if this solution addresses my needs.

## Acceptance Criteria

1. Value propositions section displays three columns on desktop/tablet
2. Column 1: Price tag icon + "Bayar Seperlumu, Bayar Semaumu" headline + brief description
3. Column 2: Browser/window icon + "Tanpa Instalasi, Buka Dimana Saja" headline + brief description
4. Column 3: Hands helping icon + "Didampingi Mentor" headline + brief description
5. Icons are SVG format, outlined style, 48-64px size, teal color
6. Icons use Heroicons, Lucide, or Phosphor icon set (consistent stroke weight)
7. On mobile (<768px), three columns stack vertically with proper spacing
8. Each column has centered text alignment and icon positioned above headline
9. Section has light gray background (#F3F4F6) to differentiate from hero/other sections
10. Typography follows design system: headlines 600 weight, descriptions 400 weight, Inter font
11. Section is keyboard navigable and screen reader accessible

### Tailwind Grid Layout

**Responsive Grid:**

```html
<!-- Mobile: 1 column, Tablet/Desktop: 3 columns -->
<div class="grid grid-cols-1 gap-8 md:grid-cols-3">
  <!-- Columns -->
</div>
```

**Breakpoint Behavior:**

- **< 768px (mobile):** `grid-cols-1` → Single column, stacked vertically
- **≥ 768px (md):** `grid-cols-3` → Three columns side-by-side

**Gap Spacing:**

- `gap-8` → 2rem (32px) gap between columns
- Applies both horizontal (desktop) and vertical (mobile) spacing

**Alternative Grid Classes (if different spacing needed):**

- `gap-6` → 1.5rem (24px)
- `gap-10` → 2.5rem (40px)
- `gap-12` → 3rem (48px)

### Accessibility Requirements

**Semantic HTML:**

- Use `<section>` for the value props section
- Add `role="region"` for explicit landmark
- Add `aria-label="Value Propositions"` for screen reader context
- Use `<h2>` or `<h3>` for headlines (maintain proper heading hierarchy)

**Icon Accessibility:**

- All icons are decorative (text provides full context)
- Add `aria-hidden="true"` to all icon components
- Screen readers will skip icons and read headlines + descriptions

**Keyboard Navigation:**

- Section should not trap keyboard focus
- No interactive elements in this component (static content)
- Users can Tab through the page without issues

**Color Contrast:**

- Headlines (navy-900 on gray-100): >7:1 ✓
- Descriptions (gray-600 on gray-100): >4.5:1 ✓
- All text meets WCAG 2.1 Level AA standards

**Screen Reader Testing:**

- Section announces as "Value Propositions region"
- Each column reads: Icon (skipped) → Headline → Description
- Content is logical and understandable without visual context
