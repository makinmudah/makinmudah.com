# Story 1.4: Problem and Solution Section with Visual Demos

## Status

Draft

## Story

**As a** visitor,
**I want** to see my specific problems acknowledged and solutions demonstrated visually,
**so that** I trust that Makin Mudah understands my pain points and can actually solve them.

## Acceptance Criteria

1. Section displays two problem-solution pairs in stacked layout (problem → solution → visual demo)
2. Problem 1: "IT terlalu ribet dan bikin pusing?" headline with empathetic description
3. Solution 1: "Solusi kami dibuat sederhana, bahkan untuk yang gaptek sekalipun" with explanation
4. Visual demo 1: GIF or static image showing simple UI (placeholder acceptable for MVP)
5. Problem 2: "Takut terjebak biaya mahal?" headline with description
6. Solution 2: "Bayar Seperlumu, Bayar Semaumu" with pricing flexibility explanation
7. Visual demo 2: Graphic showing flexible payment options (monthly/yearly/PAYG visualization)
8. CTA button "Intip Solusinya" displayed after second solution, navigates to `/catalogue-apps`
9. CTA button uses teal background, white text, rounded corners, accessible (44x44px min)
10. Section responsive: stacks cleanly on mobile, side-by-side text+visual on desktop (if space allows)
11. Images/GIFs are optimized (<500KB each), lazy-loaded for performance
12. All Indonesian language, casual tone maintained

### Image Handling

**Image Location:** `public/images/`

**Supported Formats:**

- PNG - Good for graphics with transparency
- JPG/JPEG - Good for photos
- WebP - Best compression, modern browsers (recommended)
- GIF - For animated demos (use sparingly, large file sizes)

**Image Optimization:**

- Target: <500KB per image (AC: 11)
- Use WebP format for 25-35% better compression than PNG/JPG
- Tools: TinyPNG, Squoosh, ImageOptim
- Dimensions: Max 1200px width for full-width images

**Lazy Loading:**

- Use native `loading="lazy"` attribute on `<img>` tags
- Supported in all modern browsers
- Automatically defers loading until image is near viewport
- Add width/height attributes to prevent layout shift (CLS)

**Placeholder Images for MVP:**

- AC: 4 states "placeholder acceptable for MVP"
- Can use:
  - Simple solid color rectangles with text overlay
  - Generated placeholder services (placeholder.com, via.placeholder.com)
  - Basic mockup screenshots from design tools
- Replace with real visuals in future iterations

**Image Attributes (Required):**

```html
<img
  src="/images/filename.png"
  alt="Descriptive text for screen readers"
  width="600"
  height="400"
  loading="lazy"
  class="h-auto max-w-full rounded-lg shadow-md"
/>
```

### Button Styling

**CTA Button Specifications (AC: 8, 9):**

```vue
<NuxtLink
  to="/katalog/aplikasi"
  class="inline-block rounded-lg bg-teal-500 px-8 py-3 text-base font-semibold text-white transition-colors hover:bg-teal-600"
>
  Intip Solusinya
</NuxtLink>
```

**Button Classes Breakdown:**

- `inline-block` - Allows padding and sizing
- `rounded-lg` - Rounded corners (0.5rem / 8px)
- `bg-teal-500` - Teal background (#14B8A6)
- `px-8 py-3` - Padding: 2rem horizontal, 0.75rem vertical (exceeds 44x44px min)
- `text-base` - 16px font size
- `font-semibold` - 600 weight
- `text-white` - White text
- `transition-colors` - Smooth color transition on hover
- `hover:bg-teal-600` - Darker teal on hover

**Accessibility:**

- Minimum 44x44px tap target (AC: 9) - achieved with px-8 py-3
- High contrast: white on teal meets WCAG 2.1 AA (4.5:1+)
- Keyboard accessible via NuxtLink (focusable, Enter to activate)
- Screen reader accessible (reads link text and destination)

[Source: docs/ui-architecture.md#8-styling-guidelines]

### Performance Considerations

**Image Optimization:**

- Target: <500KB per image (AC: 11)
- Recommended: <200KB for better performance
- Use WebP format (25-35% smaller than PNG/JPG)
- Provide width/height to prevent CLS (Cumulative Layout Shift)

**Lazy Loading:**

- Native `loading="lazy"` defers image loading until near viewport
- Reduces initial page load time
- Improves Largest Contentful Paint (LCP) metric

**Total Page Weight Target:**

- Landing page: <500KB (including images, fonts) - from Story 1.7 AC
- With 2 images at <500KB each, need to ensure images are actually <200KB each
- Consider compression and WebP format

**Future Optimization (Story 1.7):**

- Nuxt Image module for automatic optimization
- Responsive images with srcset
- Blur-up placeholders
- Advanced lazy loading strategies
